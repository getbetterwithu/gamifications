# 4단원 문제 도형 — PNG → SVG 전환 (보류)

**결정일**: 2026-05-12
**상태**: 보류 — 4단원 시나리오 코드화 + 게임 완성 후 별도 세션에서 작업

## 배경
현재 4단원 문제 도형 8장은 PDF에서 영역 캡처해 PNG로 저장한 상태 (`unit4_tri_quad/games/stage1_quadrilaterals/problems/*.png`). 사용자의 원래 요구는 "PDF 그림과 완전히 같아야 한다"였고, 캡처 PNG는 글자 그대로 이 조건을 만족하지만 SVG로 직접 그리는 게 더 적합하다는 데 사용자와 합의됨.

**Why SVG로 갈아타는가**:
- 벡터 — 확대/고해상도 디스플레이에서도 선명
- 게임 폰트(KaTeX 포함)와 시각 일관성
- 도형 안 라벨도 KaTeX 수식체 적용 가능
- 새로 그린 거라 저작권 깔끔
- 수정·재활용 용이

**Why now가 아닌가** (사용자 판단):
- 시나리오부터 코드화해서 게임 전체 흐름을 먼저 확인해야 SVG 디테일 조정 방향도 정확히 잡힘
- SVG 전환은 도형만의 독립 작업이라 나중에 빼서 작업 가능
- 현재 PNG가 임시 자산으로 충분히 작동

## How to apply
4단원 시나리오 코드화가 완료된 후, 사용자가 시작 신호를 줄 때 이 작업 진입.

## 작업 범위 (시작할 때 참고)
- 도형 8~12개 SVG로 새로 그리기 (도형 1개당 5~15분 예상, 총 3~5시간)
- 대상 PNG 목록: `unit4_tri_quad/games/stage1_quadrilaterals/problems/` 내 `*_figure.png`
- 라벨 위치·각도 표시·평행사변형 빗변 각도 등 디테일 정확히
- 기존 PNG는 백업·참조용으로 남기되, 게임 코드에서는 SVG로 교체

## 주의
- 이 작업이 시작될 때 사용자에게 "SVG 전환 작업 진입합니다"라고 명시
- PNG 파일은 삭제하지 말고 참조용으로 유지 (백업 폴더 등으로 이동만)
