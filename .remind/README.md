# `.remind/` — Phase 전환 시점 리마인드 시스템

> 시밀러(Claude)가 **작업 phase 전환을 감지**할 때마다 이 폴더를 확인하고, 사용자에게 적절한 항목을 자동으로 제안합니다.

---

## 🎯 무엇

좋은 아이디어가 떠올랐지만 "지금은 다른 작업 중이라 보류"한 항목들을 모아두는 곳입니다. 시밀러가 적절한 시점에 다시 꺼내서 사용자한테 묻습니다.

**메모리(`memory/`)와 다른 점**:
- `memory/` = 시밀러가 **항상** 알아야 할 사실/맥락 (timeless)
- `.remind/` = 시밀러가 **특정 시점에** 사용자한테 꺼낼 액션 아이템 (forward-looking)

---

## 🔁 동작 흐름

```
[세션 시작]
  ↓ SessionStart 훅 발동
.remind/*.md 내용 시밀러 컨텍스트에 자동 주입 (시밀러가 '인지' 상태로 시작)
  ↓
대화 진행 중...
  ↓
시밀러가 사용자 메시지에서 "phase 전환" 신호 감지
  예: "이제 이미지 작업 시작하자" / "문제 본문 채우자" / "Stage 2로 넘어가자"
  ↓
주입된 .remind/ 항목 중 trigger 조건 매칭되는 것 찾음 (status: pending 만)
  ↓
사용자한테 짧게 제안:
  "phase 전환 감지했습니다. 그 전에 [remind 항목] 한번 검토해보실까요?"
  ↓
사용자 응답:
  - "지금 처리하자" → 작업 진행 후 status: done 으로 업데이트
  - "지금 말고" → 그대로 두고 다음 트리거까지 보류
  - "취소" → status: cancelled 로 변경
```

---

## 📐 Phase 전환 = 무엇인가

**감지 대상 (예시)**:
- ✅ 시나리오 작업 → 이미지 작업
- ✅ 이미지 작업 → 문제 본문 작성
- ✅ 문제 작성 → 게임 코드 구현
- ✅ Stage 1 → Stage 2
- ✅ Unit 5 → Unit 6 (단원 전환)

**감지 안 함**:
- ❌ 같은 phase 내 자잘한 작업 완료 (예: 1번 배경 받음 → 2번 배경 받음)
- ❌ 세션 시작 자체
- ❌ 단순 질문/답변

판단은 **시밀러의 맥락 분석**에 의존합니다 (기계적 규칙으로는 감지 어려움).

---

## 📄 파일 형식

각 항목은 별도 .md 파일. 파일명: `{번호}_{slug}.md` (예: `001_image_gen_automation.md`)

```markdown
---
id: 001
title: 짧은 제목
created: YYYY-MM-DD
trigger:                          # 이 중 하나라도 충족되면 사용자한테 리마인드
  - "다음 단원 시작 (unit6 이상)"
  - "캐릭터 추가 일러스트 작업 시작"
priority: medium                  # high / medium / low
status: pending                   # pending / done / cancelled
---

## 무엇
[한 줄 요약]

## 왜 (지금이 적기인 이유)
[리마인드 시 사용자한테 제시할 논리]

## 추천안
[구체적 행동 제안 — 사용자가 "처리하자" 했을 때 시밀러가 바로 따라갈 수 있게]
```

---

## ➕ 새 항목 추가 방법

사용자 또는 시밀러가 "이 좋은 아이디어 나중에 다시 꺼내자" 결정하면:

1. `.remind/{다음번호}_{slug}.md` 파일 생성
2. 위 형식 따라 frontmatter + 본문 작성
3. **memory에는 저장하지 말 것** — 중복됨

---

## ✅ 항목 완료 시

해당 .md 파일의 frontmatter에서 `status: pending` → `status: done` 으로 변경.
삭제하지 않음 (이력 유지).
